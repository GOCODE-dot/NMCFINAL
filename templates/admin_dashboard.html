Reset & Clear Operations

Clear Old Meal Data

Delete meal orders and verified payments older than 7 days. Same as the manager's weekly clear.

Reset All Meal Orders

Delete ALL meal orders, payments, and cash requests. Student account locks will be cleared. Students remain registered.

Delete All Students

Remove all registered students AND their meal data. Manager accounts are preserved.

Full System Reset

Wipe everything — all students, orders, payments, all managers. MGR001 is re-created fresh with the default credentials from your environment (MANAGER_PASSWORD / MANAGER_BKASH). Use only for a fresh start.

Reset Manager Password

Reset MGR001's password to a value you specify. Use this if the manager is locked out.

Generate One-Time Reset Code

Generate a temporary reset code for any manager. Give this code to the manager — they enter it on the Reset Manager Password page along with their Manager ID. The code expires in 7 days.

Add Additional Manager (Keep Existing Active)

Adds a new manager account without deactivating current managers. Up to 4 managers can be active at once. Use Transfer Manager below to replace all current managers.

Transfer Manager Role (Admin Override)

Immediately deactivates all current managers and assigns the role to a new student. No student acceptance required — admin-forced. The student gets a 7-day temp password and must change it on first login.

This will deactivate all current managers. Use "Add Manager" above if you want to keep existing managers active alongside the new one.
Remove Rotation Manager SAFETY

Immediately deactivate a specific active manager (e.g. a rotation manager who should no longer have access). MGR001 cannot be removed here — use the Reset Manager options for that. This logs the action for audit purposes.

Clear Duty Rotation Schedule
